# Frank — Weekly Report Template

Delivered every Monday morning via Telegram.
Covers the prior 7 days. Requires at least 5 days of daily snapshots to be meaningful.
Each report answers these 6 questions. Skip any section where the answer is "nothing notable."

---

## 📈 Weekly Agent Scorecard — Week of {DATE}

### 1. Efficiency improvers
> Which agents got cheaper or leaner week over week?
- Cost delta, token delta, sessions this week vs. last
- Note what likely drove the improvement (reset, lighter use, prompt cleanup)

### 2. Agents getting more expensive — and why
> Which agents increased in cost, and what's the likely cause?
- Cost delta week over week
- Distinguish: more usage (healthy) vs. same usage costing more (waste)
- Specific likely cause where identifiable

### 3. Mature and stable agents
> Which agents are running predictably with no flags?
- Consistent cost-per-session, no bloat, no anomalies
- These need no action — just note them as healthy

### 4. Still in onboarding mode
> Which agents are still finding their footing?
- Elevated cost relative to task complexity
- Frequent session resets, prompt edits, or behavioral instability
- Expected to normalize — no alarm, just visibility

### 5. Highest-impact workflow changes for next week
> If [OPERATOR] changes one or two things, what saves the most tokens?
- Specific and actionable
- Ranked by estimated impact
- No vague advice

### 6. Session Health Dashboard
> Quick pulse on session state across all agents.

For each agent, report:
- Session age (days since last reset)
- Estimated cold-load size (from last snapshot cacheWrite if available)
- Last reset: ✅ fired / ⚠️ failed / ❓ unknown
- Status: 🟢 Healthy / 🟡 Watch / 🔴 Act

Thresholds:
- Session age >2 days = 🟡; >4 days = 🔴
- Cold-load >100KB = 🟡; >200KB = 🔴
- Consecutive reset failures >1 = 🔴

### 7. Agents due for review
> Which agents should have their prompts, memory, or model settings looked at?
- Flag if: SOUL.md or HEARTBEAT.md is large and possibly stale
- Flag if: agent is on Sonnet but doing simple work
- Flag if: memory files are growing without being pruned
- Include a specific suggestion for what to review
